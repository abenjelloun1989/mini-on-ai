# N8N SMS Filter Unknown Contacts Workflow

A ready-to-use n8n workflow template that filters incoming calls to identify unknown/unsaved contacts and sends SMS notifications only to numbers not in Google Contacts, eliminating manual exclusion lists.

## What's included

- Ready-to-import n8n workflow (`workflow.json`)
- 9 connected nodes
- Setup instructions for credentials and configuration

## How to import

1. Open your n8n instance
2. Go to **Workflows** → **Import from file**
3. Select `workflow.json`
4. Follow the setup steps below

## Setup

1. Step 1: Configure Google OAuth2 credentials in n8n (Settings > Credentials > New > Google OAuth2 API). You need to enable the 'Google People API' in your Google Cloud Console and set the scopes to include 'https://www.googleapis.com/auth/contacts.readonly'.
2. Step 2: Set up a Twilio account at https://twilio.com. Obtain your Account SID and Auth Token from the Twilio Console dashboard.
3. Step 3: In n8n, create an HTTP Basic Auth credential for Twilio. Set the username to your Twilio Account SID and password to your Twilio Auth Token.
4. Step 4: In the 'Send SMS Notification' node, replace 'YOUR_TWILIO_ACCOUNT_SID' in the URL with your actual Twilio Account SID.
5. Step 5: In the 'Send SMS Notification' node, replace 'YOUR_PERSONAL_PHONE_NUMBER' with the phone number that should receive unknown caller alerts (e.g., +12125551234).
6. Step 6: In the 'Send SMS Notification' node, replace 'YOUR_TWILIO_PHONE_NUMBER' with your Twilio-provided phone number (e.g., +18885551234).
7. Step 7: Create a Google Sheet named 'Unknown Calls' with columns: Timestamp, Phone Number, Call SID, Status, SMS Sent. Copy the Sheet ID from the URL (the long string between /d/ and /edit) and replace 'YOUR_GOOGLE_SHEET_ID' in both Google Sheets nodes.
8. Step 8: Configure Google Sheets OAuth2 credentials in n8n and assign them to both 'Log Unknown Call to Sheet' and 'Log Known Call to Sheet' nodes.
9. Step 9: Copy the webhook URL generated by the 'Incoming Call Webhook' node (visible after activating or test-running the workflow). Configure your phone provider (e.g., Twilio, Vonage, Plivo) to POST incoming call data to this webhook URL.
10. Step 10: Verify the incoming call payload field names from your phone provider. Update the 'Extract Caller Number' node's expression if your provider uses different field names (e.g., 'From', 'caller_number', or 'phone') for the calling party's number.
11. Step 11: Test the workflow by clicking 'Test Workflow' in n8n and sending a POST request to the webhook URL with a sample payload like: {"body": {"From": "+1XXXXXXXXXX", "CallSid": "TEST123"}}.
12. Step 12: Once verified, activate the workflow by toggling the 'Active' switch to ON in the n8n workflow editor.

## Requirements

- n8n (self-hosted or n8n.cloud)
- Relevant credentials configured in n8n (see setup steps above)
